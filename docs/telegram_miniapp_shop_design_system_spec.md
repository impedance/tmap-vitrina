## 2) MVP Telegram Mini App: что именно делаем

Цель MVP: «витрина + корзина + оформление заказа» в формате Mini App, чтобы:
1) покупатель быстро нашёл шоколад, добавил в корзину, оформил заказ;
2) вы могли показывать прототип клиентам/покупателям и тестировать спрос.

Техническая рамка Mini Apps:
- Цвета/темы берём из **themeParams** (CSS variables `--tg-theme-*`). citeturn2search1
- Учитываем **viewport/safe areas** (особенно нижние панели Telegram). citeturn2search2
- Для заказа/оплаты в Telegram можно использовать Bot Payments (физические товары) или Stars (digital). citeturn8search0turn8search1
- Для передачи заказа боту используем `Telegram.WebApp.sendData(...)` (или ваш API). citeturn8search4

## 3) Информационная архитектура (экраны)

### Навигация (простая, MVP)
Bottom-nav (3–4 пункта):
- **Каталог**
- **Корзина**
- **Заказы** (опционально)
- **Профиль** (опционально)

### Экраны
1. **Каталог**
   - Поиск (строка).
   - Фильтры: «Коллекция», «Особенность», «Вид».
   - Сортировка: популярное/новинки/цена.
   - Сетка карточек 2 в ряд (mobile-friendly).
2. **Товар**
   - Галерея фото.
   - Название + подзаголовок.
   - Цена.
   - Переключатель веса (40/60/80/1кг… если есть).
   - Stepper количества (+/−) и CTA «В корзину».
   - Аккордеоны: Состав / Хранение / Описание / Доставка.
   - «Похожие»/«Из коллекции».
3. **Корзина**
   - Список товаров: превью, название, параметры, цена, stepper, удалить.
   - Итог: сумма, доставка, контакт.
   - CTA «Оформить».
4. **Оформление**
   - Контакт: телефон/имя (можно подтянуть из Telegram user).
   - Доставка: самовывоз/курьер + адрес/комментарий.
   - Оплата: (а) «Ссылка на оплату»/инвойс, (б) «Оплата при получении» — для MVP.
   - Финальный CTA «Подтвердить заказ».
5. **(Опц.) Админ-режим**
   - Список товаров + редактирование (название, цена, активность, фото).
   - Доступ по роли (по Telegram user_id) или секретному параметру.

## 4) Дизайн-система (tokens + компоненты)

### 4.1 Принципы
- 100% поддержка **light/dark** через Telegram themeParams. citeturn2search1
- Минимум визуального «шума»: акцент на карточки товара и CTA.
- 8pt grid, крупные тач-таргеты (44px+).

### 4.2 Design Tokens (предложение)
#### Цвета (берём из Telegram)
- `--color-bg` = `var(--tg-theme-bg-color)`
- `--color-surface` = `var(--tg-theme-secondary-bg-color)`
- `--color-text` = `var(--tg-theme-text-color)`
- `--color-hint` = `var(--tg-theme-hint-color)`
- `--color-link` = `var(--tg-theme-link-color)`
- `--color-primary` = `var(--tg-theme-button-color)`
- `--color-on-primary` = `var(--tg-theme-button-text-color)`
- `--color-danger` = `var(--tg-theme-destructive-text-color)` (если доступно)
citeturn2search1

#### Типографика
- Font family: `system-ui, -apple-system, Segoe UI, Roboto, Arial`.
- Размеры: 12 (caption), 14 (body), 16 (body+), 20 (h3), 24 (h2).
- Межстрочный: 1.3–1.5.

#### Радиусы
- S: 10
- M: 14
- L: 18 (карточки)

#### Отступы
- 4 / 8 / 12 / 16 / 24

#### Тени (очень мягкие, только для карточек на светлой теме)
- `shadow-1`: 0 1 6 rgba(0,0,0,0.08)

### 4.3 Компоненты (MVP-набор)
1) **AppHeader**
- Название раздела, иконка поиска/фильтров.
2) **SearchBar**
- Input + clear + placeholder (цвет hint).
3) **FilterChips / FilterSheet**
- Chips для быстрых фильтров + bottom sheet для расширенных.
4) **ProductCard**
- Превью (ratio 1:1), название (2 строки), подзаголовок (1 строка), цена, вес.
- Primary action: tap по карточке → экран товара.
- Secondary: маленькая кнопка «+» (quick add) — опционально.
5) **PriceTag / Badge**
- «Новинка», «Vegan», «Без сахара», «Award».
6) **Segmented control** (вес/вариант)
- 40/60/80/1кг, если доступно.
7) **Stepper**
- − 1 + (в карточке товара и в корзине).
8) **PrimaryButton**
- Используем `button_color/button_text_color`. citeturn2search1
9) **CartItem**
- Превью, название, параметры, stepper, удалить.
10) **OrderSummary**
- Подытог/доставка/итого.
11) **Accordion**
- Состав/Описание/Хранение.
12) **Toast/Snackbar**
- «Добавлено в корзину», «Ошибка сети».
13) **EmptyState**
- «Ничего не найдено», «Корзина пуста».

### 4.4 Состояния и правила
- Loading: skeleton в карточках.
- Error: retry.
- Disabled: кнопки/stepper при 0 или при out-of-stock.
- Long names: 2 строки + ellipsis.
- Цены: формат `###₽`, вес: `40г`, `1кг`.

### Артефакты на выходе (обязательные)
1) **Figma file**:
   - Page 01 Foundations (colors/typography/spacing/radius)
   - Page 02 Components (варианты+состояния)
   - Page 03 Screens (Каталог / Товар / Корзина / Оформление)
2) **Design tokens export** (JSON) + mapping на CSS `--tg-theme-*`
3) **Иконки** (SVG) и правила использования
4) **Прототип** (кликабельный flow): Каталог → Товар → Корзина → Оформление

### Детализация (обязательные элементы)
- Сетка: mobile 360–390px, 2 колонки в каталоге, адаптация под 320/414.
- Bottom sheet для фильтров: коллекция/особенность/вид + кнопки reset/apply.
- Компоненты должны иметь авто-лейаут и корректные constraints.
- Контентные ограничения: названия до 60–80 символов, описание длинное.
- Доступность: контраст, тач-зоны 44px, состояния focus/pressed.

### Критерии готовности
- Любой экран должен «переключаться» между light/dark без ручной перекраски.
- Все компоненты имеют: default / pressed / disabled / loading (где нужно).
- Для каталога: сценарий «20+ карточек» не ломает верстку.

## 6) Что можно «упростить» для супер-быстрого MVP
- Фильтры сделать одним bottom sheet без чипов.
- Не делать «Заказы» (только подтверждение и отправка данных боту через sendData).
- Оплату вынести «вручную» (заявка) и добавить Bot Payments во 2 итерации. citeturn8search0
