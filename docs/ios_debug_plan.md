# План отладки и запуска Telegram Mini App на iOS

**Вводная:**
Проблема: Приложение (tmap-vitrina) открывается в Telegram Desktop, но зависает на этапе загрузки на iOS устройствах.
Стратегия: Двигаемся от проверки инфраструктуры к глубокой отладке среды выполнения телефона, локализуя проблему методом исключения.

---

## Этап 1: Проверка production-сборки (Исключаем Vite Dev Server)
**Цель:** Понять, виноват ли процесс локальной разработки (HMR сокеты Vite, сырые ES-модули) или проблема фундаментально в коде React/Telegram SDK.
**Исполнитель:** AI Agent / Junior

**Шаги:**
1. Открыть терминал в директории `client`.
2. Выполнить команду производства финальной сборки: `npm run build`.
3. Убедиться, что терминал не выдал ошибок компиляции TypeScript, и папка `dist` создана успешно.
4. Запустить предпросмотр готовой сборки: `npm run preview`.
5. Зафиксировать порт, на котором поднялся preview-сервер (по умолчанию Vite использует `4173`).
6. Запустить Cloudflare Tunnel, прокинув трафик на этот порт (например: `cloudflared tunnel --url http://localhost:4173`).
7. Обновить URL Web App в BotFather на новую выданную ссылку.
8. Полностью закрыть Telegram на iPhone (выгрузить из памяти) и открыть приложение заново.

**Ожидаемый результат и ветвление:**
- **Заработало:** Проблема в связке "Vite Dev Server <-> Cloudflare". Для релиза ваш текущий код готов. Для продолжения комфортной локальной разработки переходим к Этапу 2.
- **Не заработало (остался белый экран):** Проблема глубже (код, зависимости или строгие правила WebKit). Пропускаем Этап 2 и сразу переходим к Этапу 3.

---

## Этап 2: Замена Cloudflare Tunnel на ngrok для разработки
**Цель:** Проверить, ломает ли Cloudflare загрузку `.tsx` и подключение к WebSocket (HMR), подменяя заголовки или минифицируя файлы "на лету" (Rocket Loader).
**Исполнитель:** AI Agent / Junior

**Шаги:**
1. В директории `client` запустить стандартный сервер разработки: `npm run dev` (порт по умолчанию `5174`).
2. В отдельном окне терминала запустить ngrok: `ngrok http 5174`.
3. Скопировать выданный ngrok `https` URL.
4. Зарегистрировать этот URL в BotFather.
5. Открыть приложение в Telegram на iOS.

**Ожидаемый результат:**
- Если приложение мгновенно загрузилось — значит Cloudflare Tunnels (с вашими текущими настройками) вмешивается в трафик dev-сервера так, что iOS WKWebView его блокирует.
- *Решение:* Для локальной разработки TMA использовать исключительно ngrok или localtunnel. Cloudflare применять только для production-деплоев.

---

## Этап 3: Глубокая отладка через Safari Web Inspector
**Цель:** Конкретизировать проблему. Вместо гадания "почему белый экран?", получить точную ошибку из консоли браузера на самом телефоне.
**Исполнитель:** Junior (при наличии iPhone и Mac) / Middle developer

**Шаги:**
1. **На iPhone:** Зайти в `Настройки -> Safari -> Дополнения` и включить `Веб-инспектор`.
2. **На Mac:** Запустить Safari, зайти в `Настройки -> Дополнения` и поставить галочку `Показывать меню "Разработка" в строке меню`.
3. Соединить iPhone и Mac кабелем. (При запросе на телефоне нажать "Доверять этому компьютеру").
4. Запустить проект локально (`npm run dev` + любой туннель), убедиться, что ссылка актуальна в боте.
5. На iPhone запустить Mini App в Telegram и оставить экран с зависшей загрузкой открытым.
6. На Mac в верхнем меню Safari нажать вкладку `Разработка -> [Имя Вашего iPhone] -> [Название страницы вашего Web App]`.
7. В открывшемся окне инспектора перейти на вкладку **Console** (Консоль) и **Network** (Сеть).
8. Сделать скриншоты или скопировать красные логи ошибок (особенно ошибки касающиеся `Content-Type`, `CSP`, `CORS`, `Failed to load resource` или `SyntaxError`).

**Ожидаемый результат:** 
Текст ошибки ясно укажет на виновника. Если задачу решает AI Agent, джуну нужно просто передать агенту скопированные из Safari логи — агент мгновенно напишет решение.

---

## Этап 4: Поднятие изолированного эталона (Clean Room Test)
**Цель:** Безусловная проверка инфраструктуры. Если логи Safari непонятны или их невозможно получить (например, разработка идет на Windows/Linux) — собираем минимальный официальный рабочий шаблон с нуля.
**Исполнитель:** AI Agent

**Шаги:**
1. В терминале выйти в корневую папку рабочих проектов (рядом с папкой `tmap-vitrina`).
2. Запустить генерацию официального шаблона от разработчиков SDK:
   `npx @telegram-apps/create-mini-app@latest debug-tma-template` (в процессе выбрать React -> TypeScript -> Vite).
3. Перейти в созданную папку: `cd debug-tma-template` и установить модули: `npm install`.
4. Запустить локальный сервер: `npm run dev`.
5. Прокинуть полученный порт через ngrok.
6. Привязать новый ngrok URL к боту и протестировать на iPhone.

**Ожидаемый результат и ветвление:**
- Если внешний шаблон **загрузился успешно**: Наша инфраструктура (интернет, телефон, Telegram, ngrok) полностью исправны. Проблема в конфигурации `tmap-vitrina` (например, версии библиотек, плагины в `vite.config.ts`, конфликтующие стили). Решение: агент будет поочередно переносить компоненты из `tmap-vitrina` в чистый эталон, пока приложение не сломается, чтобы найти дефектный кусок кода.
- Если внешний чистый шаблон **тоже не загружается**: Проблема на уровне сетевого провайдера, особенностей кэша данного iPhone или глобального бага конкретной сборки клиента Telegram.
