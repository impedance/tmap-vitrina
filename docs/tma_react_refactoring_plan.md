# План рефакторинга TelegramProvider (React SDK)

## Проблема
Текущая реализация `TelegramProvider` использует ручной вызов методов `@telegram-apps/sdk` (`init`, `mountMiniApp` и др.) внутри `useEffect`. Это приводит к ошибкам вроде `ConcurrentCallError` при двойном рендеринге в React 18 (Strict Mode) и усложняет управление жизненным циклом. Кроме того, это не соответствует бест-практикам, которые требуют использования официального враппера `@telegram-apps/sdk-react`.

## Шаги решения

### 1. Подготовка и зависимости
*   Проверить наличие `@telegram-apps/sdk-react` в `client/package.json`.
*   Установить пакет, если он отсутствует (`npm install @telegram-apps/sdk-react`).

### 2. Рефакторинг `client/src/components/TelegramProvider.tsx`
*   **Удалить:** Ручные вызовы `init()`, `mount*()`, `bind*CssVars()` и `useEffect` с функциями очистки (`cleanupRef`).
*   **Добавить:** Обернуть содержимое провайдера в официальный `<SDKProvider acceptCustomStyles>`.
*   **Адаптировать UI fallback:** Логика показа плашки "Browser Mode" (через проверку окружения) должна быть аккуратно сохранена. Для использования хуков состояния SDK (например, `useMiniApp`) может понадобиться вынести эту логику во внутренний компонент, который рендерится внутри `SDKProvider`.

### 3. Настройка монтирования и мок-окружения
*   **Файлы `main.tsx` и `telegramAdapter.ts`:** Убедиться, что вызов `initTelegramAdapter()`, который создает мок-окружение для локальной разработки, выполняется строго **до** рендеринга `<SDKProvider>`. Это предотвратит ошибку `LaunchParamsRetrieveError` при работе в обычном браузере.

### 4. Рефакторинг работы с хуками
*   Пройтись по остальным компонентам приложения и заменить (где это необходимо) прямые импорты из `@telegram-apps/sdk` на официальные React-хуки: `useMiniApp()`, `useThemeParams()`, `useViewport()`, `useMainButton()` и т.д.
*   Убедиться, что перед вызовом хуков компоненты дожидаются инициализации SDK (отсутствие `undefined`), чтобы избежать падений интерфейса при асинхронной загрузке.

### 5. Тестирование и верификация
*   **Browser Mode (Локально):** Запустить приложение в десктопном браузере. Убедиться в отсутствии ошибок в консоли, корректной инициализации мок-данных и наличии плашки-индикатора Browser Mode.
*   **Telegram Client (iOS/Android):** Открыть приложение через бота прямо в Telegram. Убедиться, что ошибка `ConcurrentCallError` окончательно устранена, белый экран не появляется, а приложение корректно подхватывает тему (Day/Night) и безопасные зоны из клиента Telegram.
