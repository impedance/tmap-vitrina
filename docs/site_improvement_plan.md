# План доработки сайта Amazing Chocolate (структура, таксономия, конверсия)

Ниже — консолидированный план улучшений по итогам анализа скрейпа и структуры сайта. Фокус: исправить критичные потери (404/каноникализация), упростить навигацию (MECE), усилить “покупательские” сценарии и повысить конверсию. Большинство шагов реализуемо базовыми средствами CMS; сложные пункты помечены отдельно.

---

## Этап 0 (P0): Каноникализация URL и устранение 404 (критично)
**Цель:** Убрать потери трафика/конверсии из‑за битых ссылок и разъезда URL, улучшить индексирование и корректность аналитики.

1. **Выбрать единый канонический формат URL** (со слешем на конце или без) и применить его везде.
2. **Настроить 301‑редиректы** между вариантами (например, без `/` → с `/`) для всех страниц каталога/товаров.
3. **Привести внутренние ссылки и sitemap к канону** (никаких ссылок “в неверном формате”).
4. **Добавить/проверить `rel=canonical`** на страницах категорий/товаров (в рамках выбранного канона).
5. **Минимальный контроль качества:** прогон краулера/сканера после фикса и мониторинг 404 (Search Console/логи).

---

## Этап 1: Разделение потоков B2C (розница) и B2B (профессионалы)
**Цель:** Убрать “шум” из розничной витрины и повысить удобство заказа для кондитеров/оптовиков.

1.  **Создание нового корневого раздела:**
    *   Добавить в главное меню новый пункт: **«Для профессионалов»** (англ. *«For Professionals»* или *«Ingredients»*).
2.  **Миграция товаров:**
    *   Перенести все фасовки от 1 кг и более (например, шоколадные дропсы/каллеты) из общего каталога шоколада в новый раздел.
    *   Перенести сырье/ингредиенты (например, какао‑бобы, какао‑масло) из розничных разделов в этот же раздел.
3.  **Настройка отображения (если поддерживает шаблон):**
    *   В новом разделе включить режим списка/таблицы (вместо крупных плиток фото) с возможностью быстрого указания количества и кнопкой добавления в корзину для каждого ряда (ускоряет оптовые закупки).

---

## Этап 2: Реорганизация таксономии и фильтров (в первую очередь “Chocolate”)
**Цель:** Сделать навигацию логичной (MECE): категории = тип продукта, атрибуты = фильтры, маркетинг = подборки/бейджи.

1.  **Упрощение дерева категорий (Sub-categories):**
    *   Оставить в качестве вложенных категорий только **базовые типы продукта**:
        *   Горький шоколад (Dark)
        *   Молочный шоколад (Milk)
        *   Белый шоколад (White)
        *   Шоколад с начинками (With filling)
        *   Специальные коллекции (если нужно — отдельной категорией, но без смешения с диетами/статусами)
2.  **Настройка фасетных фильтров (Боковое меню / Чекбоксы):**
    *   Вынести свойства товара в фильтры (минимальный обязательный набор):
        *   **Тип:** Dark / Milk / White (если тип уже выбран категорией — фильтр опционален).
        *   **Диетические особенности:** Vegan, Без сахара, (если есть) Без глютена.
        *   **Происхождение какао:** Перу, Эквадор, Куба и т.д.
        *   **Процент какао:** До 50%, 50–70%, 70%+.
        *   **Начинка:** С начинкой / Без начинки (или тип начинки, если в данных это уже есть).
        *   **Формат/назначение:** Плитка / Каллеты (drops) / Для кондитеров (если не вынесли в B2B полностью).
        *   **Вес/фасовка:** 80–100 г, 200–500 г, 1 кг+ (или ваши реальные интервалы).
3.  **Вывод маркетинговых тегов из навигации:**
    *   Удалить из дерева категорий пункты “Новинки” (Newest) и “Победители премий” (Winners).
    *   Вместо этого:
        *   Настроить блоки/карусели на Главной (“Наши новинки”, “Award winners”).
        *   Создать посадочные страницы‑подборки (с преднастроенными фильтрами), на которые ведут баннеры.
4. **Нормализация “Superfood” и похожих разделов:**
    *   Подкатегории должны быть **типами** (например: какао‑бобы / нибсы / какао‑масло), а не одиночными SKU.
    *   Сырье/ингредиенты для профи — в “Для профессионалов” (Этап 1), в рознице оставить только “готовые к употреблению” позиции (если такие есть).

## Этап 3: Внедрение визуальных маркеров (Лейблы в карточках каталога)
**Цель:** Ускорить сканирование каталога взглядом, позволяя клиенту считывать главные преимущества товара без перехода внутрь карточки.

1.  **Настройка стикеров (Badge) поверх фото товаров:**
    *   `Новинка` / `New`
    *   `Хит` / `Bestseller`
    *   `Vegan` (зеленый листик)
    *   `0% Sugar`
    *   `🏆 Award` (иконка медали для шоколада, получившего премии)
2.  **Вывод микро-описаний в сетку каталога:**
    *   Использовать стандартное поле CMS «Краткое описание» (или аналог), чтобы вывести 2-3 слова о вкусовом профиле прямо под названием товара в листинге (например: *«Ноты: вишня, фундук, карамель»*).
3.  **Вывод ключевых атрибутов в листинг (микро‑спеки):**
    *   Показать 2–3 самых важных признака прямо в карточке товара (например: “70%”, “Перу”, “с начинкой”), чтобы уменьшить число “пустых” кликов и ускорить выбор.

---

## Этап 4: Прокачка категории «Подарки и наборы»
**Цель:** Повысить средний чек (AOV) за счет решений, ориентированных на сценарии покупки (Jobs-to-be-Done).

1.  **Переименование и создание сценариев:**
    *   Вместо общих названий ("Подарочный набор шоколада"), создать эмоциональные подкатегории на базе повода или бюджета:
        *   «Коллегам и боссу» (премиальный, строгий дизайн)
        *   «Для нее» / «Для него»
        *   «Впечатлить» (самые дорогие и большие наборы)
        *   «До 3000 руб.» / «До $50» (удобно для быстрого корпоративного или символического подарка)
2.  **Оптимизация Корзины / Чекаута:**
    *   Добавить в корзину сопутствующий товар (Cross-sell): опцию *"Добавить подарочную упаковку"* и *"Добавить открытку"*, с возможностью оставить текстовый комментарий, который вы распечатаете на открытке.
3.  **Посадочные страницы под сценарии (рекомендуется):**
    *   Сделать подборки “Подарок за X”, “Для коллег”, “Для нее/него” как отдельные страницы (технически — сохраненные фильтры/коллекции), чтобы вести туда из баннеров/рекламы.

## Рекомендуемый порядок внедрения (Roadmap)

1.  **✅ P0 (1–3 дня, критично):**
    *   Каноникализация URL + 301‑редиректы + приведение sitemap/внутренних ссылок к канону (Этап 0).
2.  **✅ Быстрые победы (1 неделя):**
    *   Бейджи + микро‑описания/микро‑спеки в листинге (Этап 3).
    *   “Подарки”: сценарные подкатегории + упаковка/открытка в корзине (Этап 4.1–4.2).
    *   Посадочные “Новинки/Премии” (без изменения дерева категорий) (Этап 2.3).
3.  **✅ Очистка витрины (2 неделя):**
    *   “Для профессионалов” + перенос 1кг+ и сырья/ингредиентов (Этап 1).
4.  **✅ Структурная реорганизация (3–4 неделя):**
    *   Перенастроить дерево/фильтры для “Chocolate” и нормализовать “Superfood” (Этап 2). Потребует ручного назначения атрибутов товарам в админке.
5.  **✅ Полировка (по мере готовности данных):**
    *   Привести атрибуты к единому справочнику (страны, проценты, диеты, форматы) и закрепить правила заполнения для новых SKU, чтобы фильтры не “расползались”.
